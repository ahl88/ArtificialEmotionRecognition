Created by a few engineering students with the help of Interactions LLC, this artificial emotion recognition project was created in an effort to teach automated customer feedback systems to understand human emotion. Using machine learning, we did just this by developing a program that can determine emotion from speech and text. We were far from the first team of people to work on a project of this kind, so we tried to differentiate ourselves by improving upon the artificial intelligence models of others and providing users with an easy-to-use website that gives them clear results fast. This was completed as a part of the Rutgers School of Engineering’s Senior Design Capstone Project.

  • Q: How does all this work?
    A: It all starts here on this web interface where audio is captured from the user, either through a microphone or a sound clip you upload as a file. The audio file is then sent to Google’s Speech-to-Text engine which converts your audio clip into text. Then our machine-learning algorithm uses the audio file, the text file, and a voice database to determine the emotion conveyed in the clip. Finally, the results are relayed back to the web interface where you can see what percentage of each specific emotion is understood from your input.


  • Q: What kind of applications does this project have?
    A: We believe that our project would be appropriate for a variety of applications, especially those that want to analyze the way users react emotionally to something. We originally set out to create something that would work in a customer service context. The idea was that if automated customer feedback systems had a way to understand human emotions, then they could be less frustrating to use. If the system detected anger, maybe it will say something to calm the user down or connect the user with a real person, who can more easily diffuse the situation. Our system would be applicable in this kind of scenario.


  • Q: What kind of challenges were faced during the development of this project?
    A: We struggled somewhat with putting the individual parts of our project together. It was particularly tough to combine the speech and text emotion recognition models into a single model. It was also hard for us to find the right database of audio clips to train our models, as the databases play a tremendous role when it comes to the accuracy of the speech recognition module.


  • Q: What language was this project coded in?
    A: We used HTML, CSS, and JavaScript for the web interface, Python for the machine learning side of things. Check out our code on GitHub!


  • Q: What are the future plans for this project?
    A: We are considering polishing our work and implementing it as an open API for all to use independently of this website. So, look forward to that!
